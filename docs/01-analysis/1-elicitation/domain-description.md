# Описание домена и ограничения

> Этап 1. Артефакт 1 проекта: согласованное описание домена + список ограничений.
> Заполняется после ответов заказчика ([customer-questions.md](customer-questions.md)).

## Описание домена

**Бизнес.** «Глина» — небольшая гончарная мастерская в центре города, работающая
третий год. Мастерская проводит **групповые мастер-классы** по гончарному делу.
Занятие длится 2–2.5 часа и проходит по одной из **программ**: **лепка для новичков**
(занятия за столами, без гончарного круга) или **работа на гончарном круге** для тех,
кто уже пробовал. Каждое занятие ведёт **мастер**
(сейчас их четверо, в сезон нанимаются помощники).

**Процесс.** Владелица заранее формирует **расписание на неделю вперёд**: для каждого
занятия задаются дата и время начала, программа, назначенный мастер и число мест.
В мастерской **10 гончарных кругов**; для **лепки для новичков** берут не более **6 человек**
(занятия за столами, не все круги заняты); для **работы на гончарном круге** — до **10**
(по числу кругов). Прокатный фонд — **8 комплектов** (фартук + инструменты) напрокат
(предположение аналитика, уточняется у заказчика).
Клиент видит свободные **слоты** и **записывается**, указывая, придёт он со своими
инструментами и фартуком или возьмёт **напрокат**. До занятия клиент может **отменить**
запись — при ранней отмене место освобождается, поздняя отмена оставляет место и
заготовленную глину простаивающими. Сейчас запись ведётся вручную (Instagram-директ +
ежедневник), что приводит к двойным записям и путанице, особенно в выходные.

**Цель клиентского приложения.** Дать клиентам самостоятельную онлайн-запись на
мастер-классы (просмотр и фильтрация слотов, бронирование, выбор проката, отмена,
свои бронирования), исключив ручную путаницу и двойные брони. Управление расписанием
и работу мастеров/владелицы обеспечивает **существующая инфраструктура**; приложение
потребляет её данные через API.

**Ключевые объекты домена:**

- **Мастер-класс / слот** — конкретное групповое занятие: дата/время начала, программа, мастер, всего и свободно мест. Создаётся в существующей инфраструктуре; клиенту доступен только для чтения.
- **Программа** — вариант занятия (лепка для новичков / гончарный круг), определяет длительность и потолок мест (справочные данные, read-only).
- **Мастер** — ведёт занятие; для клиента — справочные данные слота (read-only).
- **Комплект инструментов** — фартук и инструменты; может выдаваться напрокат из прокатного фонда мастерской.
- **Запись (бронь)** — резерв места клиентом на конкретный слот: число мест, своё/прокат на каждое место, статус.
- **Клиент** — записывается на занятия, отменяет запись.

## Пользователи и роли

> **Скоуп текущей работы — только клиентское мобильное приложение и API для него.**
> Роли Мастер и Владелец/Администратор обслуживаются **существующей инфраструктурой**
> (бэкенд, расписание, админка) и в этот скоуп не входят.

| Роль | Описание | Цели | В скоупе |
| :-- | :-- | :-- | :-- |
| **Клиент** | Гость мастерской, участвует в мастер-классах. Пользуется с телефона. | Увидеть и отфильтровать свободные слоты, записаться (себя и гостей), выбрать своё/прокат, видеть свои бронирования, отменить запись, получать напоминания. | **Да** |

> Мастер и Владелец/Администратор (Марина) взаимодействуют с системой через **уже реализованные**
> интерфейсы существующей инфраструктуры. Клиентское приложение лишь потребляет подготовленные
> ими данные (слоты, программы, мастера) через API.

## Ограничения

**Бизнес-ограничения**

- Всего **10 гончарных кругов** — общий потолок инвентаря.
- Новичковая группа — **не более 6 человек**; программа на круге — до 10 (по числу кругов).
- Лимит мест слота = `min(свободные места, потолок программы)`.
- **Политика отмен:** бесплатная отмена не позднее чем за **2 часа** до начала (см.
  [решение аналитика](#решения-аналитика-r-xxx) по вопросу №7); поздняя
  отмена (<2 ч) фиксируется без освобождения места и без денежных штрафов.
- **Оплата офлайн** (наличные / перевод на карту) — приложение лишь фиксирует запись и
  показывает цену.
- **Период списка слотов по умолчанию — 7 дней (R-027).** При отсутствии расписания —
  **empty state** («Пока нет доступных занятий»).
- **Отмена занятия мастерской / по форс-мажору (R-008).** При отмене организатором или
  из-за поломки оборудования (печь, электричество) бронь получает статус **«Отменено
  мастерской»** с указанием **причины** в UI (**Must**); push-уведомление — **Should**
  (в mock — симуляция). Повторная запись на слот запрещена.

**Технические / платформенные**

- **Mobile-first**: клиентское приложение рассчитано на телефон (использование в мастерской).
- Клиентское приложение взаимодействует с **существующим бэкендом через API** (в учебном проекте — mock-репозитории); слоты, программы и мастера доступны только для чтения.
- **Граница серверной интеграции (R-004).** Бэкенд — **black-box источник истины**; гарантия «**0 двойных броней**» на его стороне.
- **Данные (R-015).** Учебный проект, легаси-данных нет. Каноническая схема — **контракт API**.

**Проект**

- **Срок** — к началу осеннего сезона, ~2 месяца.
- **Бюджет** ограничен → MVP-подход.

**Границы скоупа**

| В скоупе клиентского приложения | Вне скоупа |
| :-- | :-- |
| Регистрация/вход клиента (OTP по телефону) — **Must** | Роли Мастера и Владельца, админ-экран |
| Просмотр и фильтрация списка слотов | Создание и редактирование расписания |
| Самостоятельная запись, в т.ч. несколько мест | Оценка мастера — **Phase 2** |
| Выбор «своё / прокат», контроль лимита мест и прокатного фонда | Онлайн-оплата; программа лояльности |
| Список своих бронирований, отмена с правилом 2 часов | Push-**напоминания** — Should; Won't при нехватке времени |
| Цена и фиксация записи (оплата офлайн) | |
| Статус «Отменено мастерской» + причина в UI (R-008) — **Must** | |
| Push об отмене мастерской — **Should** (mock: симуляция) | |

## Глоссарий

| Термин | Определение |
| :-- | :-- |
| **Мастер-класс / слот** | Запланированное групповое занятие: дата/время начала, программа, мастер, всего и свободно мест. |
| **Программа** | Вариант занятия: **лепка для новичков** (потолок 6, столы) или **работа на гончарном круге** (до 10). |
| **Группа** | Состав клиентов, записанных на один слот. |
| **Комплект (инструменты + фартук)** | Прокатный или собственный набор участника. |
| **Прокат** | Выдача инструментов и фартука из фонда мастерской. |
| **Своё** | Клиент приходит со своими инструментами; занимает место в группе, но не прокатный фонд. |
| **Запись (бронь)** | Резерв одного или нескольких мест на конкретный слот. |
| **Отмена** | Ранняя (≥2 ч) освобождает место; поздняя (<2 ч) фиксируется без освобождения. Мастерской — статус «Отменено мастерской». |
| **MVP** | Минимально жизнеспособный продукт к старту сезона. |

## Решения аналитика (R-xxx)

| ID | Решение | Обоснование |
| :-- | :-- | :-- |
| R-001 | Оценки мастеров — **Phase 2** | Заказчик называет функцию важной (бриф), но для учебного MVP с scope «только клиент» не блокирует запись; перенос зафиксирован **аналитиком**, не подтверждён интервью с Мариной. |
| R-013 | По гостям — только кол-во мест и прокат, без имён | Аналог референса «Волна»; ответственность за гостей на клиенте. |
| R-021 | Порог «ранней отмены» — **2 часа**, не 10 минут | В брифе «10 минут» — **пример боли** (глина и место простаивают), не формальное правило. Аналитик предложил **2 часа**: даёт мастерской время перераспределить глину и место; согласовано с референсом «Волна». **Требует подтверждения заказчиком** на следующей итерации. |
| R-022 | `Idempotency-Key` при создании брони | Защита от дублей при сбое сети. |
| R-028 | Только роль «Клиент» в текущей поставке | Из уточнения брифа. |
