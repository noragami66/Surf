# User stories

> Этап 2. Пользовательские истории. Формат: «Как <роль>, я хочу <действие>, чтобы <ценность>».
> Каждой истории — критерии приёмки.
>
> **Скоуп: клиентское приложение и API для него** (роль «Клиент»). Функции мастера и
> владельца обеспечиваются существующей инфраструктурой и здесь не описываются.

| ID | Как (роль) | Хочу (действие) | Чтобы (ценность) | Критерии приёмки | FR |
| :-- | :-- | :-- | :-- | :-- | :-- |
| US-1 | Клиент | зарегистрироваться по имени и номеру телефона без пароля | быстро начать пользоваться без барьеров | Указав имя и телефон, получаю доступ; повторный вход — по номеру телефона; сложный пароль не требуется. | FR-1, FR-2 |
| US-2 | Клиент | видеть список предстоящих слотов мастер-классов | выбрать подходящее занятие | По умолчанию показываются слоты на **7 дней** (`only_available=false`), отсортированные по времени начала; слот показывает дату/время, программу, мастера, цену, всего/свободно мест; заполненные и отменённые слоты остаются в списке с пометкой «Мест нет» и неактивной CTA; при отсутствии слотов — empty state «Пока нет доступных занятий». | FR-9 |
| US-3 | Клиент | фильтровать список слотов | быстро находить подходящий вариант | Могу фильтровать по дате/периоду, программе (**лепка для новичков** / **работа на гончарном круге**), наличию свободных мест, мастеру; внутри группы — «ИЛИ», между группами — «И»; дефолт — без фильтров; активные фильтры отображаются. | FR-38 |
| US-4 | Клиент | открыть карточку слота со всеми параметрами | понять детали перед записью | Карточка показывает дату/время, программу, мастера, длительность (2–2.5 ч), цену, всего/свободно мест, доступность прокатных комплектов, адрес мастерской. | FR-9a |
| US-5 | Клиент | записаться на выбранный слот | забронировать место на мастер-классе | После подтверждения создаётся запись; свободные места уменьшаются; запись появляется в «Моих бронированиях». | FR-10 |
| US-6 | Клиент | при записи указать своё или прокатное снаряжение | прийти со своим инвентарём или взять напрокат | Можно выбрать «своё» / «прокат» для каждого места; «прокат» уменьшает прокатный фонд, «своё» — нет. | FR-11, FR-14 |
| US-7 | Клиент | одной записью забронировать несколько мест (себя + гости) | записать компанию за раз | Можно указать 1–3 места при наличии свободных; для каждого места выбирается вариант проката. | FR-12 |
| US-8 | Клиент | быть защищённым от записи сверх лимита | не прийти на занятое место | Лимит `max_seats = min(free_seats, program.capacity_cap, 3)`; прокат: `rental_count ≤ free_rental_kits`; при гонке — без овербукинга. | FR-13, FR-15 |
| US-9 | Клиент | видеть список своих бронирований | контролировать предстоящие и прошедшие занятия | Вижу записи со статусом, параметрами слота, числом мест и вариантом проката. | FR-35a |
| US-10 | Клиент | отменить свою запись до начала | освободить место, если планы изменились | Отмена только целиком; ранняя (≥2 ч) освобождает места и прокат; поздняя (<2 ч) — «поздняя отмена», места не освобождаются, штрафов нет. | FR-16, FR-17, FR-18 |
| US-11 | Клиент | видеть цену занятия | понимать стоимость заранее | В карточке и при подтверждении показана цена; оплата офлайн, запись фиксируется без онлайн-оплаты. | FR-30 |
| US-12 | Клиент | получать напоминание о предстоящей записи | не забыть и не пропустить занятие | Push за 24 ч и 2 ч до начала — **Should** для MVP; если не успеваем — откладываем (Won't). | FR-33 |
| US-16 | Клиент | видеть, что занятие отменено мастерской, с указанием причины | перепланировать время и не ехать зря | В «Моих бронированиях» и в карточке слота — статус «Отменено мастерской» + текст причины (напр. «Сломалась печь»); CTA записи на этот слот неактивна; push об отмене — Should (в mock — симуляция). | FR-46, FR-46a |
| US-13 | Клиент | просматривать ранее загруженные данные офлайн | ориентироваться при нестабильной связи | Офлайн — кэш с пометкой «данные могли устареть»; мутации офлайн недоступны. | FR-9, FR-35a |
| US-14 | Клиент | быть защищённым от двойной брони при сбое сети | не получить дубль | `Idempotency-Key` при создании брони; повтор с тем же ключом не создаёт дубль. | FR-10, FR-22 |
| US-15 | Клиент | видеть понятные сообщения об ошибках | понять причину и что делать | Единая матрица ошибок: нехватка мест/проката, двойная бронь, отменённый слот и т.д. | FR-10, FR-16 |

> **Вне скоупа.** Истории мастера и владелицы, а также **оценки мастеров** обеспечиваются
> существующей инфраструктурой или переносятся в **Phase 2** (R-001).
