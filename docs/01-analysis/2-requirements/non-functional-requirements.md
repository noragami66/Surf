# Нефункциональные требования

> Этап 2. Как система должна работать: производительность, надёжность, безопасность, UX.
> **Скоуп — клиентское мобильное приложение.**

| ID | Категория | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- | :-- |
| NFR-1 | UX | Mobile-first: крупные элементы, высокий контраст, минимум шагов (использование в мастерской). | Высокий | бриф |
| NFR-2 | UX | Запись за ≤3 экранов до подтверждения. | Высокий | customer-questions №19 |
| NFR-3 | UX | Лёгкая регистрация: имя + телефон. | Высокий | FR-1 |
| NFR-4 | Платформа | Flutter-приложение iOS/Android (учебный проект — mock API). | Высокий | R-002 |
| NFR-6 | Производительность | Список слотов и подтверждение записи — < 2–3 с при нормальной сети. | Высокий | бриф |
| NFR-8 | Надёжность | Исключение двойных броней и овербукинга (критическая боль заказчика). | Must | BR-1 |
| NFR-9 | Надёжность | Согласованность при параллельных операциях и сбоях. | Высокий | бриф |
| NFR-11 | Безопасность | ПДн (имя, телефон) защищены; маскирование в UI и логах; токены — secure storage (R-025). | Высокий | FR-1, FR-48 |
| NFR-12 | Безопасность | Клиент видит только свои записи. | Высокий | R-028 |
| NFR-15 | Проект | Запуск ~2 месяца. | Must | BR-7 |
| NFR-16 | Проект | MVP-подход; «навороты» отложены. | Высокий | бриф |
| NFR-21 | Производительность | p95 списка слотов < 2.5 с; createBooking < 1.5 с. | Высокий | R-018 |
| NFR-24 | Offline | Просмотр кэша офлайн с пометкой; мутации офлайн запрещены. | Средний | US-13 |
| NFR-25 | A11y | Touch-target ≥ 44pt; контраст WCAG AA; labels для screen reader на ключевых экранах MVP (slots, booking, auth). | Средний | R-029 |

## Архитектурные NFR (учебный проект)

| ID | Категория | Требование | Приоритет |
| :-- | :-- | :-- | :-- |
| NFR-ARCH-1 | Архитектура | Clean Architecture: `I_Repo → RepoImpl → I_Service → ServiceImpl → BLoC → Widget`. | Must |
| NFR-ARCH-2 | Архитектура | BLoC не обращается к Repository напрямую; Widget без бизнес-логики. | Must |
| NFR-ARCH-5 | Application layer | Каждая feature содержит `application/` с `I*Service` и `*ServiceImpl`; BLoC зависит только от `I*Service`. | Must |
| NFR-ARCH-3 | Данные | Mock-репозитории как источник данных на этапе разработки. | Must |
| NFR-ARCH-4 | Локализация | UI-строки в l10n (ru). | Must |
