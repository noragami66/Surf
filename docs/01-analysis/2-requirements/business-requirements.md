# Бизнес-требования

> Этап 2. Зачем бизнесу продукт: цели, метрики успеха, ограничения верхнего уровня.

> **Скоуп — клиентское мобильное приложение и API для него.**

| ID | Требование | Приоритет | Источник |
| :-- | :-- | :-- | :-- |
| BR-1 | Устранить ручную путаницу и двойные брони при записи через Instagram-директ и ежедневник. Учёт записей — в единой системе без конфликтов по местам. | Must | Бриф (письмо Марины); domain-description |
| BR-2 | Дать клиентам самостоятельную онлайн-запись на мастер-классы («люди сами записываются, а я только смотрю»). | Must | Бриф; domain-description |
| BR-5 | Снизить простой мест и заготовленной глины из-за поздних отмен за счёт окна отмены из API и фиксации поздних отмен («ограничить, но не злить»). | Must | Бриф; R-021 |
| BR-7 | Запустить продукт к началу осеннего сезона (~2 месяца) при ограниченном бюджете, начав с MVP. | Must | Бриф; customer-questions №19 |
| BR-8 | На старте оплата офлайн (наличные / перевод); продукт фиксирует запись и показывает цену. Онлайн-оплата — за рамками скоупа. | Must | Бриф; customer-questions №9–10 |
| BR-9 | Клиентское приложение — мобильное (использование с телефона в мастерской); данные из API. | Must | Бриф; customer-questions №18 |

## Метрики успеха

| ID | Метрика | Целевой ориентир | Связь |
| :-- | :-- | :-- | :-- |
| M-1 | Доля онлайн-записей клиентами самостоятельно | ≥ 70% к концу первого сезона | BR-2 |
| M-2 | Число двойных броней / конфликтов по местам | 0 | BR-1 |
| M-4 | Доля поздних отмен от записанных | Снижение, ориентир ≤ 10% | BR-5 |
| M-6 | Готовность к старту сезона | Запуск в срок: авторизация, слоты, запись, отмены, статус «Отменено мастерской» | BR-7 |

## Ограничения верхнего уровня

- **Срок и бюджет.** ~2 месяца; MVP-подход.
- **Оплата офлайн на старте.**
- **Mobile-first** для клиента.
- **Вне скоупа:** мастер/владелец, CRUD расписания, онлайн-оплата, лояльность, **оценки мастеров — Phase 2** (R-001; [открытый риск №1](../1-elicitation/domain-description.md#открытые-риски)).
