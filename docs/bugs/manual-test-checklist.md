# Manual test checklist — «Глина» MVP

> Автоматически покрыто unit/bloc-тестами (PLATF-016, PLATF-020): 47 tests.
> Отметьте пункты после ручного прогона `cd app && flutter run`.

## Auth (UC-5, UC-6)

- [ ] Phone → OTP `0000` → имя → попадаю на «Расписание»
- [ ] Неверный OTP → сообщение об ошибке
- [ ] Logout из шапки → экран входа
- [ ] Повторный вход — сессия восстанавливается (cold start)
- [ ] После истечения access token приложение продлевает сессию (UC-6, см. unit-тесты)

## Slots + Booking (UC-1, UC-3)

- [ ] Список слотов загружается (по умолчанию **7 дней**, ~14 занятий)
- [ ] Pull-to-refresh обновляет список
- [ ] Фильтры: лепка / гончарный круг / мастер / «Только свободные»
- [ ] Сброс фильтров возвращает полный список
- [ ] Empty state при пустых фильтрах — «Нет слотов по фильтрам»
- [ ] Заполненные слоты показывают «Мест нет»
- [ ] Tap карточки → деталь слота
- [ ] «Записаться» → форма бронирования
- [ ] Подтверждение → «Мои записи» с новой карточкой
- [ ] В расписании уменьшилось число свободных мест

## My Bookings (UC-2)

- [ ] Список броней после записи
- [ ] Tap → деталь
- [ ] Отмена (ранняя) → статус «Отменена», места вернулись в слот
- [ ] После logout список пуст (новый пользователь)
